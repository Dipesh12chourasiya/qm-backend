5. Important Security Features

Never send correct answers to frontend.

Bad:

{
   question,
   options,
   correctAnswer
}

Good:

{
   question,
   options
}

Store answers only on server.

6. Anti-Cheating Features
Level 1 (Easy)

Frontend detects:

visibilitychange

If user changes tab:

document.hidden

Send violation.

POST /violations
Level 2

Disable:

copy
paste
right click

during exam.

Level 3

Fullscreen mandatory.

requestFullscreen()

If exits fullscreen:

Violation
Level 4

Webcam required.

Frontend captures frame every few seconds.

Send image:

POST /proctor/frame

Backend stores evidence.

Level 5 (Advanced)

Use:

Face API
TensorFlow
MediaPipe

Detect:

No face
Multiple faces
Looking away
Phone detected

Generate violations.

8. Future Scaling

Instead of attaching questions directly to test:

questionIds:[]

Create Question Banks.

QuestionBank

Example:

{
    _id,
    companyId,

    name:"Aptitude Set 1"
}

Then Test references:

questionBankId

and backend randomly picks:

30 questions

from 200 available questions.

This prevents users from getting identical question orders and makes cheating much harder.
